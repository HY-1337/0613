# 给下一个 AI 的项目交接总说明

本文档用于把当前恒流源项目的背景、硬件原理、已经踩过的坑、用户的新需求、后续设计方向交接给下一个 AI。

注意：不要把本文档理解成“必须沿用前一个 AI 写过的算法”。前一个 AI 写过的一些算法效果不好，尤其是复杂前馈表、自适应采样、低电流尖峰拒绝、多项式 ADC 拟合等，不建议直接照搬。

本文档的目的只是保留调试经验、硬件事实和用户后续提出的新要求。

---

## 一、项目目标

本项目是一个基于 STM32F103C8T6 的恒流源控制系统。

目标是：

1. 用户设定一个目标电流。
2. STM32 输出互补 PWM 控制外部功率驱动芯片。
3. 外部功率电路输出电流。
4. STM32 通过 ADC 检测采样电阻上的电压，计算实际电流。
5. 控制算法根据设定电流和实际电流，调整 PWM 占空比。
6. 最终让实际输出电流稳定接近设定电流。

系统还需要：

1. 旋转编码器调节设定电流。
2. LCD1602 I2C 屏显示设定电流、实际电流、占空比和提示信息。
3. 蜂鸣器用于报警或提示切换电压档位。
4. 串口用于调试、打印状态、发送命令。
5. DM3058 万用表作为真实电流参考值。

---

## 二、整体电路和信号关系

系统大致结构如下：

```text
用户输入
  ├─ 旋转编码器
  └─ 按键/串口
        ↓
STM32F103C8T6
  ├─ 根据设定电流计算控制动作
  ├─ TIM1 输出互补 PWM
  ├─ ADC1/ADC2 采样电压
  ├─ USART1 打印调试数据
  └─ I2C2 驱动 LCD1602
        ↓
功率驱动芯片
  ├─ HIN
  └─ LIN
        ↓
功率输出级
        ↓
负载 + 采样电阻
        ↓
ADC 反馈到 STM32
```

其中：

- TIM1_CH1 接驱动芯片 HIN。
- TIM1_CH1N 接驱动芯片 LIN。
- 两路 PWM 是互补 PWM。
- 采样电阻上的电压用于计算输出电流。
- DM3058 串入负载回路，作为真实电流标准。

## 五、前一个项目调试经验

### 5.1 DM3058 必须作为真实电流

前一个调试过程中发现，STM32 ADC 算出来的 `IMEAS` 不一定可靠。

因此后续拟合、校准、判断准确度时，必须以 DM3058 测到的电流为准。

正确关系应是：

```text
DM3058 实际电流 = 真值
STM32 ADC 数据 = 被校准对象
STM32 IMEAS = 根据 ADC 计算出的估计值，不是真值
```

### 5.2 ADC 精度必须先解决

如果 ADC 不准，后面所有闭环控制都会被错误反馈带偏。

之前出现过这种情况：

```text
实际电流已经很高，
但 STM32 ADC 估算电流仍然偏低，
于是控制器继续加占空比，
最终导致过冲甚至发散。
```

因此新项目必须先做 ADC 校准，再做电流控制。

### 5.3 低电流段最难

低电流段，例如 1-10 mA，问题最明显：

1. ADC 原始值可能非常小。
2. ADC 分辨率和噪声影响很大。
3. 占空比一点点变化，对相对误差影响很大。
4. 普通 PID 容易抖动。
5. 单次占空比步进太大时容易跳过目标。

所以低电流档必须单独处理，不能和高电流档共用同一套参数。

### 5.4 前馈表不能盲目依赖

之前曾经测过“占空比 -> 实际电流”的关系，并尝试做前馈表。

但是这个关系高度依赖：

1. 输入电压。
2. 负载。
3. 采样电阻。
4. 功率电路状态。
5. 板子布线。

当负载变化或输入电压档位变化时，旧前馈表可能失效。

所以以后可以使用前馈表，但必须：

1. 分档位建立。
2. 分硬件条件建立。
3. 只作为初始估计。
4. 不允许绕过安全检查。

### 5.5 之前一些算法效果不好

以下内容曾经尝试过，但不建议直接照搬：

| 尝试内容 | 问题 |
|---|---|
| 全范围一条 ADC 拟合曲线 | 不同电流段规律不同，容易局部失真 |
| 三次/二次多项式全局拟合 | 可能某段拟合好，另一段更差 |
| 低电流自适应 64 点采样 | 响应变慢，用户反馈效果更差 |
| 低电流尖峰拒绝 | 逻辑复杂，可能误判真实变化 |
| 低电流启动占空比表 | 特定硬件条件有效，换条件后不可靠 |
| 直接靠 PID 全范围控制 | 跨档位或大跳变时容易失控 |

这些经验只说明“曾经试过且效果不好”，不是不能再研究，而是不能不加判断地照搬。

## 六、用户新提出的核心控制思路

用户现在希望新的控制逻辑按以下顺序：

1. 先保证 ADC 精准。
2. 用户设定电流后，先根据档位和设定值调整 PWM，占空比进入目标电流的大概范围。
3. 用校准后的 ADC 算出当前实际电流。
4. 比较实际电流和设定电流：
   - 如果实际电流小于设定值，就适当调高占空比。
   - 如果实际电流超过设定值，就调低占空比。
5. 每次调整前，应等待 ADC 读数稳定。
6. 调整幅度由三个因素决定：
   - ADC 实际电流与设定电流的差值大小。
   - 差值比例。
   - 当前设定电流大小。
7. 当实际电流和设定电流误差小于 1% 时，停止大幅调整，让占空比稳定下来。

用户的核心想法不是“直接用复杂 PID 硬拉”，而是：

```text
先校准 ADC
再分档位安全逼近
最后用轻量控制维持稳定
```

## 七、四个电压档位需求

用户现在将输入电压分成四个档位。

| 档位 | 设定电流范围 | 输入电压 Vin | 说明 |
|---|---:|---:|---|
| L1 | 0-10 mA | 3.3 V | 极低电压，提高小电流分辨率 |
| L2 | 10-100 mA | 12 V | 占空比处于较好线性区 |
| L3 | 100-300 mA | 24 V | 安全和效率平衡 |
| L4 | 300-500 mA | 32 V | 接近大电流区，必须保留裕量 |

每个档位应单独配置：

1. ADC 校准曲线。
2. 占空比上下限。
3. 初始占空比估计。
4. 最大步进。
5. 稳定判定阈值。
6. 控制参数。
7. 安全保护阈值。

## 八、ADC 分段校准要求

用户明确提出：

1. 不一定每个档位只用一条线性曲线。
2. 同一档位内也可以继续分多个小区间。
3. 每个小区间可以用不同线性关系拟合。
4. 目标是让 ADC 校准尽量精准。

推荐形式：

```text
在某个小区间内：
真实电流 = k * ADC估算电流 + b
```

或者：

```text
真实电流 = k * ADC原始值 + b
```

选择哪种方式，需要根据实际数据决定。

低电流段建议采点更密：

```text
L1: 0, 1, 2, 3, 4, 5, 6, 8, 10 mA
```

中高电流可以相对稀疏：

```text
L2: 10, 15, 20, 30, 40, 60, 80, 100 mA
L3: 100, 120, 150, 180, 220, 260, 300 mA
L4: 300, 340, 380, 420, 460, 500 mA
```

## 九、跨档位切换提示需求

当用户从一个设定电流改到另一个设定电流，并且目标电流属于不同电压档位时，系统必须提示用户切换输入电压。

例如：

```text
原来 Set = 150 mA，属于 L3，需要 24 V
现在 Set = 8 mA，属于 L1，需要 3.3 V
```

这时系统必须：

1. LCD 显示提示。
2. 蜂鸣器响一声。
3. LCD 明确显示目标电压。
4. 暂停闭环控制。
5. PWM 输出降为 0 或安全占空比。
6. 清空控制器内部状态。
7. 等待用户确认或检测电压已经切换完成。

LCD1602 可能不能显示中文，所以可以用英文和数字：

```text
Change Vin To
3.3V L1
```

或者：

```text
Vin -> 12V
Range L2
```

用户原始要求：

```text
当电流设定在某个档位（比如小于 10）时，如果从其他档位切换到这个小于 10 的档位，请先在 LCD 屏幕上给我一个提示，显示“请切换至”以及该档位对应的电压数值。

具体的逻辑要求如下：
1. LCD 显示提示：当你从一个档位切换到另一个档位的设定电流时，需要在 LCD 屏幕上显示提示文字。
2. 蜂鸣器提醒：蜂鸣器要响一声（滴一下），提醒我需要改变电压。
3. 显示目标数值：在 LCD 上还要明确显示出需要改变的电压值是多少。
```

## 十、必须避免跨档位失控和发散震荡

前一个项目调试中出现过严重失控现象：

```text
当设定电流从 10-20 mA 一下调到 100 mA 时，
实际电流突然从十几二十 mA 直接跳到 600 mA，
然后不断震荡，
震荡幅度甚至越来越大，
逐渐从几十变到 700、800、900 mA。
```

这种情况必须避免。

可能原因：

1. 跨档位没有暂停输出。
2. 还没切换输入电压，控制器就开始按新目标调节。
3. 旧 PID 积分没有清空。
4. 旧滤波值和历史误差继续参与计算。
5. 初始占空比不适合新档位。
6. ADC 读数还没稳定。
7. 控制周期太快。
8. 单次占空比步长太大。

强制要求：

1. 跨档位时禁止直接连续闭环。
2. 跨档位时必须先将 PWM 降到 0 或安全状态。
3. 必须清空积分、滤波、历史误差和旧控制状态。
4. 必须提示用户切换输入电压。
5. 必须等待用户确认或检测到电压正确。
6. 重新控制时必须从该档位安全初始占空比开始。
7. 若实际电流明显超过设定值，应立即减小占空比。
8. 若实际电流超过安全上限，应进入故障状态。

用户原始要求：

```text
这种情况一定要避免，你想个办法处理一下。

对，这种情况一定要避免，然后把这个要求也写入我的需求文档里面。
```

## 十一、自适应逼近控制建议

推荐控制方式不是一上来就用重 PID，而是状态机式逐步逼近。

建议流程：

```text
用户设定电流
    ↓
判断目标档位
    ↓
如果跨档位，进入电压切换提示
    ↓
确认电压正确
    ↓
给安全初始占空比
    ↓
等待 ADC 稳定
    ↓
比较校准后实际电流和设定电流
    ↓
根据误差大小、误差比例、设定电流大小调整占空比
    ↓
逐步逼近目标
    ↓
误差小于 1%
    ↓
锁定或进入轻量维持
```

调整占空比时应考虑：

| 因素 | 作用 |
|---|---|
| 绝对误差 | 误差越大，允许步子越大 |
| 相对误差 | 小电流时比例误差更重要 |
| 设定电流大小 | 小电流步长必须更小 |
| 当前档位 | 每个档位步长不同 |
| 误差方向是否反转 | 反转说明跨过目标，应减小步长 |

误差方向反转时：

```text
step = step / 2
```

## 十二、轻量 PID 的使用要求

用户提出，可以在不断逼近目标电流后，再加一个轻量 PID。

用户原始要求：

```text
我觉得这个 PID 不需要搞得很复杂，就是一个比较轻量一点的，不用有那么大负担。只要让它的响应尽量稳定，能保持结果稳定就可以。能不能实现这个效果？
```

前一个 AI 的理解：

```text
可以实现。
但是轻量 PID 不应该负责从 0 拉到目标，也不应该负责跨档位切换。
它只应该在电流已经接近目标后，做小范围稳定维持。
```

推荐策略：

1. 粗调用自适应步进，不用 PID。
2. 细调用小步逼近，不用重 PID。
3. 进入目标附近后，再启用轻量 PI/PID。
4. D 项默认不用或非常弱。
5. I 项必须限幅。
6. 跨档位、设定值大变化、故障保护时必须清空积分。
7. PID 输出只能在当前占空比附近小范围调整。

推荐定位：

```text
轻量 PID = 稳定器
不是主控制器
```

## 十三、建议控制状态机

建议把控制逻辑写成状态机：

| 状态 | 作用 |
|---|---|
| IDLE | 空闲，PWM 关闭或安全输出 |
| VOLTAGE_CHANGE_REQUIRED | 等待用户切换输入电压 |
| INIT_DUTY | 设置安全初始占空比 |
| WAIT_ADC_STABLE | 等待 ADC 读数稳定 |
| COARSE_ADJUST | 粗调，快速接近目标 |
| FINE_ADJUST | 细调，小步逼近 |
| LOCKED | 误差小于 1%，锁定占空比 |
| LIGHT_PID_HOLD | 轻量 PID 维持稳定 |
| FAULT | 过流或异常，关闭 PWM 并报警 |

每个状态必须有：

1. 进入条件。
2. 退出条件。
3. 超时保护。
4. 安全处理。

## 十六、下一个 AI 最应该注意的事

1. 不要直接沿用前一个 AI 的复杂算法。
2. 先理解电路和四个输入电压档位。
3. 先做 ADC 校准。
4. 所有校准以 DM3058 为真值。
5. 跨档位必须停机提示，不能直接连续调。
6. 大幅设定变化必须重置控制状态。
7. 轻量 PID 只能做稳定维持。
8. 低电流档必须使用更细占空比步进。
9. 代码必须模块化，不能继续堆在 `main.c`。
10. 任何控制算法都必须先考虑安全，防止电流突然冲高。

## 十七、几个具体场景示例

下面这些例子是为了让下一个 AI 更具体地理解用户想要的行为。

### 17.1 从 L1 切到 L2

当前状态：

```text
Set = 8 mA
当前档位 = L1
当前输入电压 = 3.3 V
当前电流稳定
```

用户旋转编码器，把设定值调到：

```text
Set = 50 mA
目标档位 = L2
目标输入电压 = 12 V
```

正确流程：

```text
1. 识别到 L1 -> L2 跨档位。
2. 立即暂停闭环。
3. PWM 输出降为 0。
4. 清空积分、滤波、历史误差、上一次调整方向。
5. LCD 显示：
   Vin -> 12V
   Range L2
6. 蜂鸣器滴一声。
7. 等待用户确认已经切换到 12V。
8. 重新进入 INIT_DUTY。
9. 根据 L2 的安全初始占空比开始逼近 50mA。
```

错误流程：

```text
用户刚调到 50mA，
代码直接沿用 L1 的占空比和旧 PID 状态继续调。
```

这种错误流程可能导致电流突然冲高。

### 17.2 从 L3 切到 L1

当前状态：

```text
Set = 180 mA
当前档位 = L3
当前输入电压 = 24 V
```

用户把设定调到：

```text
Set = 5 mA
目标档位 = L1
目标输入电压 = 3.3 V
```

正确流程：

```text
1. 立刻 PWM = 0。
2. 不允许继续用 24V 档控制 5mA。
3. LCD 提示切换到 3.3V。
4. 蜂鸣器短响。
5. 等用户确认。
6. 清空控制器状态。
7. 使用 L1 的极小步长和高精度 ADC 校准。
```

原因：

```text
如果还接着 24V，用 5mA 的小目标去控制，
很小的占空比变化都可能造成巨大相对误差。
```

### 17.3 设定值没有跨档位，只是同档内变化

当前状态：

```text
Set = 30 mA
当前档位 = L2
```

用户调到：

```text
Set = 60 mA
仍然是 L2
```

这种情况不需要提示切换电压。

但仍然建议：

```text
1. 判断设定变化幅度。
2. 如果变化较大，重置轻量 PID 的积分。
3. 保留当前档位。
4. 用 L2 的初始估计或当前占空比附近重新逼近。
```

### 17.4 设定值超出安全范围

用户设定：

```text
Set = 580 mA
```

虽然系统设计范围可以到 600mA 用于识别和保护，但正常输出应在 550mA 附近设置保护线。

建议处理：

```text
1. 不直接输出。
2. LCD 显示 LIMIT 或 OVER SET。
3. 蜂鸣器报警或短响提示。
4. PWM 保持 0。
```

如果实际电流检测到：

```text
I_actual > 550 mA
```

无论设定值是多少，都应：

```text
PWM = 0
进入 FAULT
蜂鸣器报警
LCD 显示 OVERCURRENT
```

## 十八、短路与小电阻负载的具体区分例子

用户特别关心：

```text
如何区分“短路”和“负载很小、设定电流又很大”？
```

不能只看电流大，也不能只看负载电阻小。

### 18.1 正常小电阻负载例子

假设：

```text
Set = 400 mA
I_actual = 398 mA
V_load = 1.2 V
R_load = 3.0 Ω
控制后可以稳定
```

虽然负载电阻比较小，电流也比较大，但它不一定是短路。

原因：

```text
1. 实际电流接近设定电流。
2. 负载电压不是 0。
3. 计算出的负载电阻稳定。
4. 控制器能稳定维持目标。
```

这应该被视为正常负载。

### 18.2 疑似短路例子

假设：

```text
Set = 100 mA
I_actual = 450 mA
V_load = 0.03 V
R_load ≈ 0.067 Ω
Duty 已经降低但电流仍偏高
```

这应该判断为疑似短路。

建议处理：

```text
1. 立即快速降低占空比。
2. 观察 50-100ms。
3. 如果电流回到安全范围，可以解除疑似状态。
4. 如果仍然异常，进入确认短路。
```

### 18.3 确认短路例子

假设：

```text
Set = 100 mA
I_actual = 650 mA
V_load ≈ 0 V
R_load ≈ 0 Ω
持续超过 50ms
```

处理：

```text
PWM = 0
蜂鸣器持续报警
LCD 显示 SHORT
等待用户检查线路并复位
```

## 十九、开路保护具体例子

开路不是简单地看“电流小”。

因为设定 1mA 时，电流本来就很小。

### 19.1 不应该判断为开路的情况

```text
Set = 1 mA
I_actual = 0.8 mA
Duty 很小
```

这不是开路。

### 19.2 应该判断为开路的情况

```text
Set = 50 mA
Duty 已经超过 L2 的有效最小占空比
I_actual 长时间接近 0
V_load 较高或接近输出端空载电压
R_load 计算为极大或无法稳定
持续 300-500ms
```

处理：

```text
PWM = 0
蜂鸣器报警
LCD 显示 OPEN
等待用户检查负载
```

## 二十、ADC 和负载电阻测量的具体说明

### 20.1 ADC 校准的实际目标

ADC 校准不是为了让 STM32 的显示“看起来好看”，而是为了让控制器得到真实反馈。

例如：

```text
DM3058 = 50.00 mA
STM32 IMEAS = 42.00 mA
```

这说明 ADC 传递函数有问题。

后续应拟合：

```text
真实电流 = f(ADC 数据)
```

而不是让 PID 去弥补 ADC 的错误。

### 20.2 分段拟合例子

不要强行用一条线覆盖 L1：

```text
0-10 mA
```

可以拆成：

```text
0-2 mA
2-5 mA
5-10 mA
```

每段单独拟合：

```text
I_real = k1 * ADC + b1
I_real = k2 * ADC + b2
I_real = k3 * ADC + b3
```

如果某一段 ADC 原始值经常为 0，说明硬件或采样方式在该段分辨率不足，应明确记录，而不是用算法强行“猜”。

### 20.3 负载电阻计算例子

假设：

```text
采样电阻 R_sense = 5.62 Ω
ADC1 校准后得到 V_sense = 0.562 V
ADC2 校准后得到 V_total = 5.562 V
```

则：

```text
I_load = V_sense / R_sense = 0.562 / 5.62 = 0.1 A = 100 mA
V_load = V_total - V_sense = 5.562 - 0.562 = 5.000 V
R_load = V_load / I_load = 5.000 / 0.1 = 50 Ω
```

注意：

如果 `I_load` 很小，比如小于 5mA 或 10mA，`R_load = V / I` 会非常敏感，参考价值会下降。

## 二十一、轻量 PID 的具体接入方式

轻量 PID 不应替代整个逼近流程。

推荐流程：

```text
INIT_DUTY
    ↓
COARSE_ADJUST
    ↓
FINE_ADJUST
    ↓
误差进入 1%-3%
    ↓
LIGHT_PID_HOLD
```

在 `LIGHT_PID_HOLD` 中：

```text
Duty = LockedDuty + P_small + I_slow
```

限制：

```text
Duty 只能在 LockedDuty 附近小范围变化
I 项有限幅
误差小于 1% 时冻结或减慢 I 项
跨档位时清空 I 项
故障时清空 I 项
```

示例：

```text
Set = 100 mA
自适应逼近后 Duty = 18.30%
I_actual = 99.4 mA
进入 LIGHT_PID_HOLD

允许 PID 微调范围：
18.10% - 18.50%
```

这样 PID 只能维持稳定，不能把占空比突然推到危险位置。

## 二十二、下一个 AI 开始工作前应先做什么

建议下一个 AI 按以下顺序：

1. 阅读本文档。
2. 阅读 `docs/ADC数据给下一个AI的说明.md`。
3. 查看 `docs/adc_duty_reference_stable_averaged.csv`，理解历史数据。
4. 不要直接复制旧 `main.c` 的全部逻辑。
5. 先规划模块结构。
6. 先实现基础模块：
   - LCD
   - 编码器
   - 蜂鸣器
   - PWM
   - ADC
7. 再实现四档电压和保护状态机。
8. 再做 ADC 校准。
9. 最后实现逼近控制和轻量 PID。
