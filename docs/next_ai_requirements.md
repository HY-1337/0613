# 下一阶段恒流源控制需求与改进要求

本文档记录从“输入电压分为四个档位”开始提出的新需求和改进方向，供下一个 AI 接手项目时使用。

## 1. 总体目标

目标是把恒流源控制逻辑改成更安全、更稳定、更容易调试的结构：

1. 先保证 ADC 测量准确。
2. 再根据设定电流选择对应输入电压档位。
3. 根据档位给出安全的初始占空比。
4. 通过稳定 ADC 读数逐步逼近目标电流。
5. 误差进入允许范围后锁定或轻量维持。
6. 严禁跨档位或大幅设定变化时直接沿用旧控制状态。

## 2. 四个输入电压档位

设定电流范围按输入电压分成四档：

| 档位 | 设定电流范围 | 目标输入电压 Vin | 设计重点 |
|---|---:|---:|---|
| L1 | 0-10 mA | 3.3 V | 极低电流，高分辨率，ADC 精度优先 |
| L2 | 10-100 mA | 12 V | 中低电流，线性区控制 |
| L3 | 100-300 mA | 24 V | 中高电流，响应与安全平衡 |
| L4 | 300-500 mA | 32 V | 高电流，安全限幅与防超调优先 |

每个档位应有独立的：

- ADC 校准曲线
- 占空比范围
- 最小/最大步进
- 稳定判定阈值
- 控制参数
- 安全保护参数

不要试图用同一条 ADC 曲线或同一套控制参数覆盖所有电流范围。

## 3. ADC 校准优先

后续控制必须以 DM3058 实测电流作为真实值。STM32 的 ADC/IMEAS 只是被校准对象。

ADC 校准要求：

1. 每个电压档位单独校准。
2. 每个档位内部可以继续分成多个小区间。
3. 每个小区间可以使用不同线性拟合，不要求一个档位只用一条线。
4. 优先使用稳定状态数据，不使用过渡点、开路点、明显异常点。
5. 对低电流档 L1，采点要更密，尤其 0-10 mA。

推荐思路：

```text
I_real = k_i * I_adc_raw + b_i
```

其中 `i` 表示当前档位内的某个小区间。

## 4. 电压档位切换提示

当用户通过编码器、按键或串口修改设定电流后，如果新设定电流所属档位和当前档位不同，系统必须进入“电压切换等待”流程。

要求：

1. LCD 显示提示。
2. 蜂鸣器短响一次。
3. LCD 明确显示目标输入电压。
4. 暂停闭环控制。
5. PWM 输出降到 0 或安全占空比。
6. 清空积分项、滤波状态、历史误差和旧占空比控制状态。
7. 等待用户确认或检测到输入电压已切换后，再重新进入控制流程。

LCD1602 如果不能显示中文，可以显示：

```text
Change Vin To
3.3V L1
```

或者：

```text
Vin -> 12V
Range L2
```

## 5. 必须避免跨档位失控和发散震荡

之前调试中出现过严重失控现象：

```text
设定电流从 10-20 mA 突然改到 100 mA 后，
实际电流直接跳到 600 mA，
随后在几十、几百、700、800、900 mA 之间发散震荡。
```

这种情况必须在新逻辑中彻底避免。

可能原因包括：

- 跨档位时没有暂停输出
- 没有切换到正确输入电压
- 旧 PID 积分没有清空
- 旧滤波值、历史误差继续参与控制
- 前馈表或初始占空比不适用于新档位
- ADC 反馈延迟或不准确
- 单次占空比步进太大
- 控制周期快于电路/ADC 稳定时间

强制保护要求：

1. 设定电流跨越 L1/L2/L3/L4 档位时，禁止直接连续控制。
2. 跨档位时必须先 `PWM = 0` 或进入安全输出。
3. 必须重置控制器内部状态。
4. 必须等待电压切换确认。
5. 重新启动时必须从该档位的安全初始占空比开始。
6. 如果检测到实际电流超过设定值过多，应立即降占空比或关断。
7. 如果电流超过绝对安全上限，应进入故障状态并报警。

建议保护阈值：

```text
如果 I_actual > max(Set * 1.5, Set + 安全裕量)
    立即大幅降低占空比或 PWM=0

如果 I_actual > 绝对安全上限
    PWM=0
    PID/控制器停止
    蜂鸣器报警
    LCD 显示 FAULT
```

## 6. 自适应逼近控制流程

用户设定目标电流后，控制流程应为：

```text
选择档位
    ↓
检查是否需要切换电压
    ↓
给安全初始占空比
    ↓
等待 ADC 稳定
    ↓
比较 ADC 校准电流和设定电流
    ↓
按误差大小、误差比例、设定电流大小调整占空比
    ↓
逐步逼近
    ↓
误差小于 1% 后进入稳定/锁定
```

占空比调整幅度应同时参考：

1. 绝对误差：`abs(Set - I_actual)`
2. 相对误差：`abs(Set - I_actual) / Set`
3. 当前设定电流大小
4. 当前档位
5. 误差方向是否反转

误差方向反转时，说明刚才跨过目标，应缩小步长，例如：

```text
step = step / 2
```

## 7. 稳定判定和锁定

不要在 ADC 还没有稳定时继续频繁调整占空比。

建议：

- 粗调阶段只要求 ADC 大致稳定。
- 细调阶段要求 ADC 更稳定。
- 误差小于 1% 后停止大幅调整。
- 连续多次误差都在允许范围内后，锁定当前占空比。
- 锁定后继续监测，如果负载变化导致误差连续超出阈值，再重新进入微调。

## 8. 轻量 PID 要求

可以在自适应逼近控制之后增加一个轻量 PID，但它不应承担大范围拉升电流的任务。

轻量 PID 的定位：

- 不是负责从 0 快速拉到目标。
- 不是负责跨档位切换。
- 只负责在已经接近目标后做小范围稳定维持。

建议使用轻量 PI 或非常弱的 PID：

```text
输出占空比 = 已逼近占空比 + 小幅 P 修正 + 慢速 I 修正
```

建议：

- D 项默认不用或非常弱，因为 ADC 噪声会被 D 项放大。
- I 项必须有限幅，防止积分饱和。
- 跨档位、设定值大幅变化、故障保护触发时必须清空积分。
- 误差小于 1% 时可以冻结积分或只允许极慢积分。
- PID 输出只能在很小范围内微调，不允许直接把占空比大幅拉高。

轻量 PID 应该是“稳定器”，不是“主控制器”。

## 9. 控制状态机建议

建议使用状态机，不要把所有逻辑写在一个循环里：

```text
IDLE
VOLTAGE_CHANGE_REQUIRED
INIT_DUTY
WAIT_ADC_STABLE
COARSE_ADJUST
FINE_ADJUST
LOCKED
LIGHT_PID_HOLD
FAULT
```

每个状态都应有明确进入条件、退出条件和超时保护。

## 10. 代码结构要求

不要再把所有逻辑堆在 `main.c`。

`main.c` 只负责：

1. HAL 初始化
2. 时钟初始化
3. CubeMX 外设初始化
4. 调用 `App_Init()`
5. 在主循环调用 `App_Loop()` 或 `App_Process()`

建议拆分文件：

```text
Core/Inc/app.h
Core/Inc/app_config.h
Core/Inc/app_adc.h
Core/Inc/app_pwm.h
Core/Inc/app_control.h
Core/Inc/app_current_range.h
Core/Inc/app_lcd.h
Core/Inc/app_uart_cmd.h
Core/Inc/app_encoder.h
Core/Inc/app_buzzer.h

Core/Src/app.c
Core/Src/app_adc.c
Core/Src/app_pwm.c
Core/Src/app_control.c
Core/Src/app_current_range.c
Core/Src/app_lcd.c
Core/Src/app_uart_cmd.c
Core/Src/app_encoder.c
Core/Src/app_buzzer.c
```

核心原则：

- `main.c` 不放业务细节。
- ADC 校准放在 ADC 模块。
- 档位逻辑放在 current range 模块。
- 控制状态机放在 control 模块。
- LCD 只负责显示。
- 串口只负责命令和调试输出。
- PWM 模块只负责安全设置占空比。

## 11. 调试数据要求

给下一个 AI 的数据表必须明确区分：

- DM3058 实际电流：真值
- 输出占空比：控制输出
- ADC 原始值/电压：被校准对象
- STM32 IMEAS：仅参考，不是真值

优先使用：

```text
docs/adc_duty_reference_stable.csv
docs/adc_duty_reference_stable_averaged.csv
```

其中 averaged 表更适合拟合，stable 表适合追溯原始稳定点。
