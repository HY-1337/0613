# 可复用代码模块与保护逻辑说明

本文档用于交接当前项目中已经基本验证可用、后续可以迁移到新工程的代码逻辑。

注意：本文档不是要求下一个 AI 原封不动复制当前 `main.c`。相反，后续应该把这些功能拆成独立模块，不要继续堆在主函数文件里。

---

## 一、后续工程建议模块划分

建议把当前可用逻辑拆成以下模块：

| 模块 | 建议文件 | 作用 |
|---|---|---|
| 应用入口 | `app.c / app.h` | 统一初始化和主循环调度 |
| 配置参数 | `app_config.h` | 电流范围、保护阈值、档位参数 |
| LCD 显示 | `app_lcd.c / app_lcd.h` | LCD1602 I2C 显示 |
| 编码器输入 | `app_encoder.c / app_encoder.h` | EC11 旋转编码器和按键 |
| 蜂鸣器 | `app_buzzer.c / app_buzzer.h` | 提示音、报警音 |
| PWM 输出 | `app_pwm.c / app_pwm.h` | TIM1 互补 PWM，占空比设置与关断 |
| ADC 采样 | `app_adc.c / app_adc.h` | ADC1/ADC2 采样、滤波、校准 |
| 电流控制 | `app_control.c / app_control.h` | 自适应逼近、轻量 PID、状态机 |
| 档位管理 | `app_range.c / app_range.h` | L1/L2/L3/L4 电压档位判断 |
| 保护逻辑 | `app_protection.c / app_protection.h` | 过流、短路、开路、过压等保护 |
| 串口命令 | `app_uart_cmd.c / app_uart_cmd.h` | 串口调试命令和状态输出 |

`main.c` 后续只应该保留：

```text
HAL_Init()
SystemClock_Config()
MX_GPIO_Init()
MX_ADC1_Init()
MX_ADC2_Init()
MX_TIM1_Init()
MX_USART1_UART_Init()
MX_I2C2_Init()
App_Init()
while (1) { App_Loop(); }
```

## 二、LCD 模块

### 2.1 当前已验证的硬件

LCD 使用：

```text
LCD1602 + PCF8574T I2C 转接板
```

常见 I2C 地址：

```text
0x27
```

当前使用 I2C2：

| STM32 引脚 | LCD 转接板 |
|---|---|
| PB10 | SCL |
| PB11 | SDA |
| GND | GND |
| 5V 或 3.3V | VCC，按 LCD 模块要求 |

当前 PCF8574T 映射：

| PCF8574 位 | LCD 功能 |
|---|---|
| P0 | RS |
| P1 | RW |
| P2 | E |
| P3 | 背光 BL |
| P4 | D4 |
| P5 | D5 |
| P6 | D6 |
| P7 | D7 |

### 2.2 当前可复用逻辑

当前 LCD 逻辑已经能完成：

1. I2C 地址扫描。
2. PCF8574T 扩展口写入。
3. LCD1602 4 位模式初始化。
4. 写命令。
5. 写数据。
6. 清屏。
7. 设置光标位置。
8. 写一整行。
9. 显示设定电流、实际电流、占空比。

后续可以保留的接口形式：

```text
LCD_Init()
LCD_Clear()
LCD_SetCursor(row, col)
LCD_WriteLine(row, text)
LCD_ShowStatus(set_current, measured_current, duty)
LCD_ShowVoltageChangePrompt(range, target_voltage)
LCD_ShowFault(reason)
```

### 2.3 后续新增 LCD 需求

当电流设定跨越电压档位时，LCD 需要提示用户切换输入电压。

例如：

```text
Vin -> 3.3V
Range L1
```

或者：

```text
Change Vin To
12V L2
```

普通 LCD1602 不一定支持中文，因此不强制用中文显示。

## 三、蜂鸣器模块

### 3.1 当前已知硬件

蜂鸣器是低电平触发。

| STM32 引脚 | 功能 |
|---|---|
| PB9 | 蜂鸣器控制 |

逻辑：

```text
PB9 = LOW  -> 蜂鸣器响
PB9 = HIGH -> 蜂鸣器不响
```

### 3.2 原本蜂鸣器逻辑必须保留

原本已经实现过：

```text
设定电流超过阈值时，蜂鸣器报警。
```

后续需求改成：

```text
设定电流范围 0-600 mA
当电流超过 550 mA 时：
1. 自动切断占空比，PWM = 0
2. 蜂鸣器开始响
3. LCD 显示报警信息
```

### 3.3 后续蜂鸣器功能建议

建议蜂鸣器支持三种模式：

| 模式 | 作用 |
|---|---|
| 短响一次 | 提示用户切换输入电压 |
| 连续报警 | 过流、短路、开路等故障 |
| 静音 | 正常工作 |

建议接口：

```text
Buzzer_Init()
Buzzer_BeepOnce(duration_ms)
Buzzer_AlarmOn()
Buzzer_AlarmOff()
Buzzer_Task_1ms()
```

## 四、编码器模块

### 4.1 当前已知硬件

编码器为 EC11，带按键。

| 编码器引脚 | STM32 |
|---|---|
| A | PB12 |
| B | PB13 |
| C / SW | PB8 |
| GND | GND |
| VCC | 3.3V 或模块要求电源 |

### 4.2 原本编码器逻辑必须保留

原本已经调通过：

1. 旋转编码器可以改变设定电流。
2. 编码器方向曾经调过，后续不要随意改方向。
3. 编码器输入需要消抖。
4. LCD 刷新不能阻塞编码器响应。

### 4.3 后续建议

编码器模块只负责产生事件，不直接控制 PWM。

建议输出事件：

```text
ENCODER_STEP_PLUS
ENCODER_STEP_MINUS
ENCODER_BUTTON_CLICK
```

由控制模块根据事件修改设定电流。

如果设定电流跨越档位，应触发档位切换逻辑，而不是编码器模块自己处理。

## 五、PWM 模块

### 5.1 当前可用逻辑

当前项目使用 TIM1 互补 PWM：

| STM32 引脚 | 功能 | 外部连接 |
|---|---|---|
| PA8 | TIM1_CH1 | HIN |
| PA7 | TIM1_CH1N | LIN |

后续 PWM 模块应负责：

1. 启动互补 PWM。
2. 设置占空比。
3. 设置最大占空比限制。
4. 紧急关断。
5. 根据档位限制最小/最大占空比。

### 5.2 关键要求

后续电流范围为：

```text
0-600 mA
```

保护要求：

```text
如果电流超过 550 mA：
    PWM = 0
    蜂鸣器报警
    LCD 显示报警
```

注意：这里的“电流超过 550 mA”应优先指实际检测电流，而不仅仅是设定值。

建议分两层：

```text
设定值 > 550 mA:
    禁止启动或提示超限

实际电流 > 550 mA:
    立即 PWM = 0，进入故障状态
```

## 六、ADC 模块

### 6.1 当前可用逻辑

当前项目中 ADC 的基本思路是：

```text
ADC1 测采样电阻相关电压
ADC2 测输出端/负载端相关电压
```

根据之前用户描述：

```text
ADC1 可以测量采样电阻两端电压，从而计算支路电流。
ADC2 - ADC1 可以得到负载端电压。
负载电压 / 电流 可以得到负载电阻。
```

### 6.2 后续必须重视 ADC 校准

ADC 是闭环控制的基础。

必须以 DM3058 实测电流为真值，校准 ADC。

后续不建议直接信任：

```text
STM32 IMEAS
```

它只是根据 ADC 算出来的估计值。

## 七、负载电阻测量方法

### 7.1 基本公式

假设：

```text
V_sense = 采样电阻电压
R_sense = 采样电阻阻值
I_load = 负载电流
V_total = 采样电阻 + 负载的总电压
V_load = 负载电压
R_load = 负载电阻
```

则：

```text
I_load = V_sense / R_sense
V_load = V_total - V_sense
R_load = V_load / I_load
```

如果沿用当前 ADC 逻辑：

```text
V_sense 来自 ADC1 校准后电压
V_total 来自 ADC2 校准后电压
V_load = ADC2电压 - ADC1电压
R_load = V_load / I_load
```

### 7.2 为什么负载电阻必须谨慎计算

当电流很小的时候：

```text
I_load 接近 0
```

此时计算：

```text
R_load = V_load / I_load
```

会非常不稳定，甚至无限大。

因此只有在电流足够大且稳定时，才计算负载电阻。

建议条件：

```text
I_load > 5 mA 或 10 mA
ADC 连续多次稳定
PWM 已经稳定
没有处于档位切换、启动、故障状态
```

### 7.3 负载电阻测量精度要求

为了让负载电阻有参考价值，需要：

1. ADC1 和 ADC2 分别校准。
2. 采样电阻阻值必须用实测值。
3. 分压比必须校准。
4. 使用稳定状态下的平均值。
5. 不在电流过小或刚改变占空比时计算。
6. 对多次计算结果做中位数或去极值平均。

建议输出：

```text
R_load_ohm
V_load_v
I_load_ma
```

LCD 上可以只显示简化结果，串口可以打印详细值。

## 八、短路保护判断方法

### 8.1 难点

用户提出的问题非常关键：

```text
如何区分“输出端短路”和“负载很小 + 设定电流很大 + 占空比很大”？
```

不能只用“电流很大”判断短路。

因为正常情况下，如果设定电流很大，负载电阻很小，系统确实可能输出较大电流。

### 8.2 不能单独使用的判断条件

以下条件单独使用都不可靠：

| 条件 | 为什么不可靠 |
|---|---|
| 电流很大 | 高设定电流时本来就可能很大 |
| 占空比很大 | 高电流或高负载时也可能需要大占空比 |
| 负载电阻小 | 小电阻负载不一定是短路 |
| ADC2 电压低 | 低阻负载时负载电压也可能低 |

### 8.3 推荐短路判断逻辑

短路应通过多个条件共同判断。

建议条件：

```text
条件 A：实际电流明显超过设定电流
条件 B：实际电流超过绝对安全阈值
条件 C：负载电压接近 0
条件 D：计算出的负载电阻低于短路阈值
条件 E：这种状态持续一段时间，而不是单次毛刺
```

例如：

```text
if I_actual > max(Set * 1.5, Set + 50mA)
and V_load < 0.2V
and R_load < R_short_threshold
and 持续超过 50-100ms:
    判断为疑似短路
```

如果再满足：

```text
I_actual > 550mA
```

则直接进入严重保护：

```text
PWM = 0
蜂鸣器报警
LCD 显示 SHORT 或 OVERCURRENT
```

### 8.4 如何区分小电阻负载和短路

关键区别是：

正常小电阻负载：

```text
实际电流接近设定电流
负载电压虽然低，但不是完全塌陷
R_load 有稳定可测的非零值
控制器可以稳定在目标附近
```

短路：

```text
实际电流远超设定电流
负载电压接近 0
R_load 接近 0
占空比很小也可能电流很大
控制器无法稳定
```

因此短路判断不要只看 R_load 小，而要看：

```text
R_load 小 + 电流失控 + 负载电压塌陷 + 持续存在
```

### 8.5 建议短路状态分级

建议分两级：

#### 疑似短路

```text
R_load 很低
V_load 很低
I_actual > Set 明显
```

处理：

```text
快速降低占空比
观察是否恢复
```

#### 确认短路

```text
I_actual > 550mA
或者降低占空比后电流仍异常
或者疑似短路持续超过一定时间
```

处理：

```text
PWM = 0
蜂鸣器报警
LCD 显示 SHORT
等待用户复位或确认
```

## 九、开路保护判断方法

### 9.1 开路现象

负载开路时：

```text
负载电阻趋近无穷大
实际电流接近 0
输出端电压可能升高
无论怎么提高占空比，电流仍然上不去
```

### 9.2 推荐开路判断逻辑

不能只用“电流小”判断开路，因为低设定电流本来就小。

建议条件：

```text
Set > 某个最小有效电流
PWM 占空比已经超过该档位的最小有效占空比
实际电流仍然接近 0
V_load 较高或接近当前档位可达到的输出电压
R_load 计算值非常大或无法稳定计算
该状态持续一段时间
```

例如：

```text
if Set > 5mA
and Duty > D_open_check_min
and I_actual < max(1mA, Set * 0.05)
and V_load > V_open_threshold
and 持续超过 300-500ms:
    判断为开路
```

处理：

```text
PWM = 0
蜂鸣器报警
LCD 显示 OPEN
等待用户检查负载
```

### 9.3 开路检测注意事项

在以下状态不要判断开路：

1. 刚启动的瞬间。
2. 刚切换输入电压档位。
3. 用户设定电流非常小，例如 0-1 mA。
4. ADC 尚未稳定。
5. PWM 还没有给到有效占空比。

## 十、0-600 mA 范围和 550 mA 保护要求

后续设定电流范围需要支持：

```text
0-600 mA
```

但保护逻辑要求：

```text
超过 550 mA 自动切断占空比并报警
```

建议区分三种情况：

| 情况 | 处理 |
|---|---|
| 用户设定值 > 550 mA | 不允许启动，LCD 提示超限，蜂鸣器报警 |
| ADC/DM3058 实际电流 > 550 mA | 立即 PWM = 0，进入过流故障 |
| 电流接近 500-550 mA | 允许用于识别和校准，但控制必须保守 |

为什么还需要支持到 600 mA：

```text
为了准确识别 500-600 mA 之间的异常状态，
以及判断是否超过 550 mA 保护线。
```

## 十一、建议保护状态

建议保护模块输出以下状态：

| 状态 | 含义 |
|---|---|
| PROTECT_NONE | 正常 |
| PROTECT_OVERCURRENT | 过流 |
| PROTECT_SHORT_SUSPECT | 疑似短路 |
| PROTECT_SHORT_CONFIRMED | 确认短路 |
| PROTECT_OPEN_LOAD | 开路 |
| PROTECT_ADC_INVALID | ADC 数据无效 |
| PROTECT_RANGE_NOT_CONFIRMED | 电压档位未确认 |

控制模块看到非正常状态后，应决定是否：

1. 降占空比。
2. 关闭 PWM。
3. 暂停闭环。
4. LCD 提示。
5. 蜂鸣器报警。
